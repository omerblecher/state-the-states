# Phase 6 — UI Design Contract: Speed Typing Mode

> This is a Flutter/Dart mobile project. All tokens are logical pixels (dp). No web framework, no shadcn, no Tailwind.

## Spacing Scale

Declared values (multiples of 4dp — matches Flutter Material 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gap in HUD row; star icon horizontal padding |
| sm | 8dp | Between chip grid items (`Wrap.spacing`); chip grid run spacing (`Wrap.runSpacing`); HUD inner padding; card list gap; input field bottom padding |
| lg | 16dp | Card inner padding; `ListView` horizontal padding; input field side padding; HomeScreen header left padding |
| 2xl | 24dp | CompletionScreen horizontal padding; score card inner padding |
| 3xl | 32dp | CompletionScreen vertical padding; major section spacing |

**Note on 20dp:** The HomeScreen header left padding is an inherited codebase pattern from prior phases. It is not introduced in Phase 6 and is not declared as part of this phase's spacing scale. Phase 6 uses 16dp for all new horizontal padding.

Exceptions:
- HUD bar height: 48dp (fixed — COPPA/A11Y touch-target floor, matches `GameHud` pattern)
- Mode icon container: 52×52dp (established `_ModeCard` pattern — slightly above tap-target minimum for visual balance)
- Primary CTA button height: 56dp (established `CompletionScreen` + `WelcomeScreen` pattern)
- Secondary CTA button height: 48dp (established `CompletionScreen` pattern)
- TextField minimum touch target: 48dp (A11Y-01 requirement)

**Source:** observed values in `home_screen.dart`, `completion_screen.dart`, `game_hud.dart`.

## Typography

Flutter `TextStyle` declarations. All sizes in logical pixels.

Maximum 4 declared sizes. Maximum 2 declared weights. AppBar title (20dp w500 system default) is system-inherited and excluded from the declared scale.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 14dp | w400 (`FontWeight.normal`) | 1.5 | HUD score/timer labels; found counter (`$matchedCount / 50`); card description text; chip grid overflow ellipsis; `_ModeCard` "Not played"; input field hint text; `_StatRow` values; "Choose a mode" subtitle |
| Input / Chip | 16dp | w400 | 1.5 | Input field entered text (`TextField` style); chip label text |
| Heading | 22dp | w700 (`FontWeight.bold`) | 1.2 | HomeScreen "State the States" title |
| Display | 28dp | w700 | 1.2 | CompletionScreen "Well done!" title; star formula header |

**Chip label weight note:** Chip labels use `w700` (not `w600`). All bold/emphasis elements use `w700` exclusively — `w600` is not in the declared weight set.

**Weight consolidation:** Two weights only — `w400` (body/label/input) and `w700` (all bold/emphasis). Former `w800` and `w600` usages are both expressed as `w700`.

**Source:** measured from `home_screen.dart`, `completion_screen.dart`, `game_hud.dart`, `welcome_screen.dart`.

## Screen Inventory

### New Screens in Phase 6

#### SpeedTypingScreen (`lib/features/typing/speed_typing_screen.dart`)

**Focal point declaration:** Primary focal point is the TextField input row (bottom-anchored, keyboard-docked) — this is where the player's attention and interaction is concentrated. Secondary anchor is the chip grid (shows cumulative progress toward 50/50). The HUD bar is a status indicator, not a focal point; it should not compete visually with the input row or chip grid.

```
Scaffold (resizeToAvoidBottomInset: true)
├── AppBar
│   ├── backgroundColor: #00695C
│   ├── foregroundColor: Colors.white
│   ├── title: Text("Speed Typing")
│   └── leading: IconButton(Icons.home → context.go('/'))
│
└── body: Column
    ├── [HUD row — 48dp fixed height]
    │   Container(height: 48, color: Colors.grey.shade800)
    │   └── Row(padding: EdgeInsets.symmetric(horizontal: 8))
    │       ├── Text("Score: $score", 14dp w700 white)
    │       ├── SizedBox(width: 8)
    │       ├── Text("$matchedCount / 50", 14dp w400 white)
    │       ├── Spacer
    │       ├── Text("MM:SS", 14dp tabularFigures white)
    │       ├── [mute toggle 48×48dp]
    │       └── [pause button 48×48dp]
    │
    ├── [Chip grid — Expanded, scrollable]
    │   Expanded
    │   └── SingleChildScrollView(padding: EdgeInsets.all(8))
    │       └── Wrap(spacing: 8, runSpacing: 8)
    │           └── Chip × matchedPostals.length
    │               ├── label: Text(stateName, 16dp w700 white)
    │               └── backgroundColor: Colors.green.shade700
    │
    └── [Input row — pinned above keyboard]
        Padding(EdgeInsets.fromLTRB(16, 0, 16, 8))
        └── TextField
            ├── textCapitalization: TextCapitalization.characters
            ├── textInputAction: TextInputAction.done
            ├── style: TextStyle(fontSize: 16)
            ├── decoration: InputDecoration(
            │       hintText: "Type a state name or code...",
            │       border: OutlineInputBorder())
            └── enabled: mapDataAsync.hasValue
```

**Empty chip grid state:** Show no chips; the `Wrap` contains zero children. No placeholder text or illustration — the visible empty area and the text field with hint text communicate "start typing" without requiring a separate empty-state widget.

**Chip grid when nearing completion:** Grid scrolls naturally. Chips at position 49/50 and 50/50 are visually identical to earlier chips — no special last-chip treatment.

## Interaction Contract

### Input Field Behavior

| Trigger | Action |
|---------|--------|
| Any character typed | Displayed UPPERCASE (via `TextCapitalization.characters`) — no manual `toUpperCase()` per keystroke |
| Enter / Done key | `_onSubmit(value)` called; field cleared immediately regardless of match result |
| Valid state name (not yet found) | `submitTyping()` returns `true`; success SFX; chip added to Wrap; field cleared |
| Valid postal code (not yet found) | Same as valid name — `GA`, `CA`, etc. accepted |
| Already-found state | `submitTyping()` returns `false`; error SFX; +5 penalty; field cleared (treated as miss, D-01) |
| Non-matching string | `submitTyping()` returns `false`; error SFX; +5 penalty; field cleared |
| Partial name (`MASS`) | Non-matching — error SFX + penalty (D-01: exact match only) |
| Missing space (`NEWYORK`) | Non-matching — error SFX + penalty (D-02: space required) |
| Backspace before Enter | No penalty — field editing is free; penalty only on Enter press |
| Empty Enter | No action, no penalty — `trimmed.isEmpty` guard returns early |
| Submit while data loading | No action — `enabled: false` on `TextField` while `stateDataProvider` is loading |
| 50th state found | Success flow, then auto-navigate to `/complete` via `addPostFrameCallback` |

### Chip Grid

| State | Visual |
|-------|--------|
| 0 chips | Empty `Wrap` — no chips, no placeholder, hint text in TextField visible |
| 1–49 chips | Chips fill left-to-right, new chip appended at end |
| 50 chips | Grid shows all 50 before navigation fires; scroll available if chips overflow |
| Chip content | Full uppercase state name (`CALIFORNIA`, `NEW YORK`) per D-06 |
| Chip color | `Colors.green.shade700` background, white text, w700 |

### Touch Targets (A11Y-01 compliance)

All interactive elements in SpeedTypingScreen must be ≥48×48dp:

| Control | Minimum Size | Implementation |
|---------|-------------|----------------|
| HUD mute toggle | 48×48dp | `SizedBox(width: 48, height: 48)` wrap (matches `GameHud`) |
| HUD pause button | 48×48dp | Same pattern |
| TextField | 48dp height minimum | `InputDecoration` default exceeds 48dp on Material |
| AppBar home button | 48×48dp | AppBar `leading` IconButton default |

### Semantics Labels (A11Y-01 compliance)

| Widget | Semantics |
|--------|-----------|
| SpeedTypingScreen Scaffold | Implicit from AppBar title |
| HUD mute button | `Semantics(label: 'Toggle sound, currently [muted/on]', button: true)` |
| HUD pause button | `Semantics(label: 'Pause game', button: true)` |
| Found-state chip | `Semantics(label: '$stateName found')` on each `Chip` |
| TextField | Default `Semantics` from `TextField` with hint text |
| Mode 5 `_ModeCard` | `Semantics(button: true, label: 'Speed Typing mode')` (matches existing `_ModeCard` pattern) |

### Audio Feedback (A11Y-02 compliance — multimodal)

| Event | Audio | Visual | Haptic |
|-------|-------|--------|--------|
| Correct submission | `audioService.playCorrect()` | New chip appears in grid | Light (inherited from existing correct-drop pattern) |
| Wrong submission | `audioService.playError()` | No visual change (field clears) | None — typing mode, no token bounce |
| Game complete | No separate SFX — navigation to CompletionScreen | CompletionScreen stars + confetti if PB | None |

## Implementation Notes for Executor

These notes are design-contract observations, not architectural decisions (see RESEARCH.md for full architecture).

1. **Countdown suppression:** `SpeedTypingScreen` must call `startGame(GameMode.speedTyping, skipCountdown: true)`. Do NOT render the numeric 3-2-1 countdown overlay — there is no map to orient to. The `_maybeStartGame()` guard pattern handles the deferred start safely.

2. **HUD adaptation:** `SpeedTypingScreen` does NOT use the `GameHud` widget directly (that widget has a progress bar and hint button for the map mode). Instead, build a compact HUD inline: `Container(height: 48, color: Colors.grey.shade800, ...)` Row with score, found counter, and timer — visually consistent with `GameHud` style without reusing its widget.

3. **Chip state source of truth:** Chips are derived from `session.matchedPostals`. The screen reads `session.matchedPostals` from `ref.watch(gameSessionProvider)` and maps postals to state names via the `stateDataProvider` states list. No local chip list is maintained in screen state.

4. **Session restore routing:** `HomeScreen.onContinue` must route `speedTyping` sessions to `/type`, not `/play`. This is a required change to `home_screen.dart`.

5. **CompletionScreen Play Again routing:** Must be mode-aware. `speedTyping` → `/type`; all others → `/play`.

6. **displayName extension:** All existing `.name` usages on `GameMode` in `CompletionScreen` AppBar and `_StatRow` must be replaced with `.displayName` to prevent "speedTyping" raw enum name rendering.
