<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SharedPreferences ↔ app | Persisted snapshot/score/mute JSON is the only data crossing into the app; on a rooted device this store is attacker-writable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Tampering | GameStateRepository.loadSession deserialization | mitigate | Any parse/validation failure (bad JSON, unknown enum name via `values.byName`) is caught → returns null AND clears the key (D-08). No exception reaches the child; no error dialog. Tested by the corrupt-snapshot case. |
| T-02-03 | Tampering | HighScoreRepository.saveBestScore | mitigate | Lower-wins guard (`score < current`) means a tampered HIGH score can never overwrite a legitimate best; the worst a tamperer achieves is writing a fake low score, which has no security consequence (single-user local game). Tested by the lower-wins case. |
| T-02-04 | Information Disclosure (COPPA) | snapshot/score/mute JSON | mitigate | Persisted fields are exclusively game-state values (phase, mode, score, elapsedSeconds, errorCount, activePostal, hintsRemaining, hintPenalty, matchedPostals, plus int high scores and bool prefs). Zero device/user/install identifiers. Verified by field enumeration. |
| T-02-SC | Tampering | package installs | accept | No package-manager installs in this phase — all deps declared Phase 1. |
</threat_model>
