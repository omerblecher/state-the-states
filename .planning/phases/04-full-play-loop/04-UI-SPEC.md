# Phase 4 — UI Design Contract: Full Play Loop

> Visual and interaction contract for the complete playable game loop.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Flutter/Material Design — no web component library applies.

| Property | Value |

## Spacing Scale

All values in logical pixels (dp). Multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, star row gaps, inline padding |
| sm | 8dp | Intra-row element gaps, progress bar spacing, card internal rows |
| md | 16dp | Card internal padding, screen horizontal padding, mode card list gaps (12dp rounding to 12 — see exception) |
| lg | 24dp | Pause overlay vertical padding, completion screen section gaps |
| xl | 32dp | Completion screen major section breaks |
| 2xl | 48dp | HUD height (fixed), hint button minimum height, all interactive control minimum tap target |
| 3xl | 64dp | not used in Phase 4 |

**Exceptions:**
- Mode card list vertical gap between cards: 12dp (midpoint between sm and md — direct port from Flags `home_screen.dart` line 263/268/274/280).
- Token tray strip height: 120dp (fixed, direct port from Flags `flag_tray.dart` container height).
- HUD height: 48dp (fixed, direct port from Flags `game_hud.dart` container height).
- All interactive controls minimum tap target: 48×48dp (COPPA/a11y requirement, Phase 5 formal audit but Phase 4 must build to this baseline — direct port from Flags `game_hud.dart` SizedBox wrappers).
- Completion screen score card internal padding: 24dp (port from Flags `completion_screen.dart`).
- Pause overlay card horizontal margin: 40dp each side (port from Flags).

## Typography

All values in logical pixels (sp). Flutter uses `sp` for font sizes; these are declared as `double` literal values in `TextStyle(fontSize: N)`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display/Heading | 28sp | w700 | 1.2 | Home screen title "State the States"; completion screen mode name in AppBar; pause overlay title "Paused"; countdown overlay seconds; completion screen "Well done!" title |
| Body | 17sp | w700 | 1.4 | Mode card name label; token face abbreviation large text; State token abbreviation (Learn, Geographical Master modes) |
| Body small | 14sp | w400 | 1.5 | Token face full state name (States Master mode); stat row labels in completion screen; "Choose a mode" subtitle |
| Label/Caption | 12sp | w400 | 1.4 | Mode card description; HUD score/timer text; tray name beneath card; mode card "Best: N" / "Not played" secondary text; privacy footer |

**Notes:**
- Two weights in regular use: w400 (regular) and w700 (bold). No other weights used.
- Token face text must fit within 90dp card width. Use `TextOverflow.ellipsis` + `maxLines: 1` for state names. Abbreviations (2 chars) never overflow.
- HUD uses `FontFeature.tabularFigures()` on the elapsed timer so digits don't shift width during play (port from Flags `game_hud.dart` line 68).

## Component Inventory

### 1. Home Screen

**Widget:** `HomeScreen` extends `ConsumerStatefulWidget` — replaces Phase 1 placeholder body.

**Focal point:** The mode card list — the 4 full-width gradient cards are the primary visual anchor.

**Layout:** `Scaffold(backgroundColor: #F0F4F8)` → `SafeArea` → `Column`:
1. **Header row** (padding: fromLTRB(20, 20, 12, 4)): map/USA icon (28dp, color `#1565C0`) + app title text 28sp w700 color `#0D2E6B` + `info_outline` IconButton (48dp tap target).
2. **Section subtitle** (padding: left 20, bottom 12): "Choose a mode" 14sp grey w400.
3. **Expanded ListView** (horizontal padding 16dp): 4 `_ModeCard` widgets separated by 12dp `SizedBox` gaps.
4. **Privacy footer** (bottom 8dp): centered TextButton "Privacy Policy" 12sp grey.

**Source:** Direct port of `FlagsRoundTheWorld/lib/features/home/home_screen.dart`. Omit: ad banner slot, `_checkSavedSession` / continue dialog (Phase 5 HOME-03), `url_launcher` About dialog (Phase 5).

### 2. Mode Card

**Widget:** `_ModeCard` — `StatefulWidget` with `SingleTickerProviderStateMixin` for press-scale animation.

**Dimensions:** Full-width container (horizontal padding from ListView), height ~80dp (determined by content — port Flags which uses `padding: all(16)`).

**Press animation:** Scale 1.0 → 0.97 over 80ms (forward) / 150ms (reverse), `Curves.easeInOut`. `GestureDetector` with `onTapDown`/`onTapUp`/`onTapCancel`.

**Decoration:**
- `BorderRadius.circular(16)`
- `LinearGradient` from mode color → `Color.lerp(modeColor, Colors.black, 0.2)`, top-left to bottom-right
- `BoxShadow`: color = modeColor.withValues(alpha: 0.4), blurRadius 12, offset (0, 4)

**Internal layout (Row, padding: all(16)):**
- Icon container: 52×52dp, `BorderRadius.circular(12)`, `Colors.white.withValues(alpha: 0.2)` background. Icon: 28dp, white, mode-specific (Learn: `Icons.explore`, States Master: `Icons.flag`, Geographical Master: `Icons.compass_calibration`, Grand Master: `Icons.emoji_events`).
- 12dp gap.
- `Expanded` Column: mode name 17sp w700 white + 4dp gap + description 12sp white alpha 0.8.
- 8dp gap.
- `FutureBuilder<int?>` Column: 3-star row (18dp `Icons.star_rounded` / `Icons.star_outline_rounded`, amber / white alpha 0.4) + 4dp gap + "Best: N" 12sp or "Not played" 12sp (white alpha 0.5).

**Empty state (never played):** "Not played" label, 0 stars shown as 3 outline stars in white alpha 0.4.

**Tap target:** Full card is tappable via `GestureDetector`. Minimum card height satisfies 48dp tap target requirement. Add `Semantics(button: true, label: '{mode name} — best score {N}, {starCount} stars')` wrapper.

**Source:** Direct port of `FlagsRoundTheWorld/lib/features/home/home_screen.dart` `_ModeCard`.

### 3. Game Screen (MapScreen extension)

**Widget:** `MapScreen` (`ConsumerStatefulWidget`) — Phase 4 extends Phase 3's production widget.

**Focal point:** The USA map canvas (InteractiveViewer child) — the entire center column draws the eye; HUD and tray are deliberately lower-contrast to keep focus on map placement.

**Overall layout:** `Scaffold` → `Stack`:
1. `GameHud` (top, 48dp height, full width) — positioned before or via `Column`.
2. `Expanded` `InteractiveViewer` area (middle, fills remaining height).
3. `StateTray` strip (bottom, 120dp height, full width).
4. Pause overlay (`Positioned.fill`) — conditionally shown.
5. Countdown overlay (`Positioned.fill`) — shown during `GamePhase.countdown` (5s countdown per Phase 2 D-01).
6. Zoom FABs (`Positioned`, bottom-right, above tray — `bottom: 128dp` to clear 120dp tray + 8dp gap).

**Preferred layout:** Use `Column` with `Expanded` for the map area rather than absolute `Positioned` for HUD/tray, to let the Flutter layout engine handle the vertical stack cleanly. This avoids manual height calculations. The `Stack` is only needed for overlays (pause, countdown) layered above the column.

**DragTarget strategy:** One large `DragTarget<String>` covering the entire `InteractiveViewer` child (the 1000×628 `SizedBox`). The `onAcceptWithDetails` callback calls `_toSceneFromGlobal(details.offset + StateTray.kPinAnchor)` to recover the drop scene coordinate, then dispatches to `stateHitTest()`. This is the Flags pattern — one `DragTarget` + hit-test dispatch, not one `DragTarget` per state. Source: 04-CONTEXT.md D-07 and Flags `map_screen.dart`.

**kPinAnchor:** `const Offset(45, 70)` — copied from Flags `FlagTray.kPinAnchor`. The drop coordinate is `DragTargetDetails.offset + kPinAnchor`. This recovers the pin tip (bottom-centre of the 90dp wide, 60dp tall card + 10dp triangle). Document in `StateTray` as a named constant.

**Countdown overlay:** Full-screen semi-transparent overlay (`Colors.black54`) shown during `GamePhase.countdown`. Displays remaining seconds as large centered text (28sp w700 white). The full-screen overlay context makes the number visually prominent without requiring a larger size class. "GO!" displayed on the final tick before `GamePhase.playing`. Dismissed automatically when phase transitions to `playing`. No user interaction possible during countdown.

**Correct drop sequence:**
1. `UsaMapPainter` receives updated `matchedPostals` — state fills grey `#AAAAAA`.
2. `OverlayEntry` fly-to-centroid animation (port from Flags `flag_sequence.dart`): token widget animates from tray position → centroid screen position over ~400ms, then fades out and removes.
3. Success SFX + light haptic.
4. `AnimatedSwitcher` in tray advances to next token (fade transition, 200ms).

**Incorrect drop sequence:**
1. `StateTray.triggerBounce()` — token bounces (500ms elasticOut, Offset(20, -10) and back).
2. Error SFX + medium haptic.
3. `ScaffoldMessenger.of(context).showSnackBar(SnackBar(content: Text('Not quite — try again'), duration: Duration(milliseconds: 1500)))`.

**Fly-to-centroid OverlayEntry:**
- Insert via `Overlay.of(context).insert(_overlayEntry)`.
- `TweenAnimationBuilder<Offset>` from tray card global position → centroid screen position, duration 400ms, `Curves.easeOut`.
- After landing: `FadeTransition` opacity 1.0 → 0.0 over 200ms.
- On complete: `_overlayEntry.remove(); _overlayEntry = null;`.
- The token widget displayed in the overlay is the same card shell as in the tray.
- Reference: Flags `flag_sequence.dart` + `highlight_painter.dart`.

### 4. State Token Card (StateTray widget)

**Widget:** `StateTray` — replaces `FlagTray`. Accepts `currentPostal`, `stateName`, `mode`, `cardKey`, `hintsRemaining`, `onHintPressed`.

**Container:** 120dp height, full width, `Colors.grey.shade200` background, horizontally centered content.

**Layout (Row, mainAxisSize: min):** hint button + 8dp gap + animated card column.

**Card dimensions:** 90dp wide × 60dp tall (`_cardShell`), plus 10dp pin triangle below (feedback only), plus 4dp gap + label text below card (when `showName` is true).

**Card decoration:** `Colors.white` fill, `BorderRadius.circular(8)`, box shadow `blurRadius: 4, offset: (2,2), color: #44000000`.

**Card face content by mode:**

| Mode | Card face | Label beneath card |
|------|-----------|-------------------|
| Learn (MODE-01) | State abbreviation, 28sp w700 centered | Full state name, 12sp w400, `TextOverflow.ellipsis` |
| States Master (MODE-02) | Full state name, 17sp w700 centered, overflow ellipsis | Full state name, 12sp w400 (mirrors Flags two-layer pattern) |
| Geographical Master (MODE-03) | State abbreviation, 28sp w700 centered | No label (`showName: false`) |
| Grand Master (MODE-04) | Solid palette color fill, no text | No label (`showName: false`) |

**Grand Master token color strategy:** Assign palette color by the state's index in the shuffled sequence: `palette[sequenceIndex % 6]`. The sequence is randomized at game-start, so the same state gets a different color each game. This gives no geographic cue from a fixed color-per-state (04-CONTEXT.md "Claude's Discretion"). The 6 palette colors are the same as the map fill colors: `[#8DB87F, #D4B483, #E8A055, #E89090, #A07EC8, #E8D870]`.

**Drag feedback widget:** Same card shell + Material elevation 6 + pin triangle (20×10dp, color `#FF6600`, `_DownTriangle` clipper). The pin tip is the drop registration point.

**dragAnchorStrategy:** `_pinAnchorStrategy` returns `StateTray.kPinAnchor` = `Offset(45, 70)`. This positions the pin tip at the pointer, not the card corner.

**childWhenDragging:** Same card shell at `Opacity(0.3)` — shows a ghost card in the tray while dragging.

**Bounce animation:** `AnimationController` 500ms, `Tween<Offset>(begin: Offset.zero, end: Offset(20, -10))`, `Curves.elasticOut`. `triggerBounce()` runs forward then reverses. Wraps the card column in `Transform.translate`.

**AnimatedSwitcher transition (token advance):** Fade transition only (`FadeTransition`). No slide or scale — slide moves the hit-test area during transition, making the Draggable unreachable (Flags Bug 1, `map_screen.dart` line 942 comment). Duration 200ms.

**Hint button:** `ElevatedButton.icon`, minimum size 88×48dp, icon `Icons.lightbulb_outline` 18dp, label "Hint ×N". Semantics label: "Hint button, N hints remaining". Pressing hint is a no-op in Phase 4 (visual wired; logic deferred to Phase 5 HINT-01). Display `hintsRemaining` from `GameSession`.

**Tray Semantics:** `Semantics(label: 'State token: {name or abbreviation}', button: false)` wraps the `Draggable`. Draggable is described as the state identity appropriate to the mode.

**Background:** `Colors.grey.shade800` (#424242).

### 6. Pause Overlay

**Trigger:** HUD pause button tap, Android back button during `GamePhase.playing` or `GamePhase.paused`.

**Layout:** `Positioned.fill` → `GestureDetector(onTap: () {})` (consumes pass-through taps) → `Container(color: Colors.black54)` → `Center` → `Card(margin: symmetric(horizontal: 40))` → `Padding(symmetric(vertical: 24, horizontal: 32))` → `Column(mainAxisSize: min)`:

1. Title "Paused" — 28sp w700.
2. 24dp gap.
3. **Resume** — `ElevatedButton`, full width, height 48dp. Label: "Resume". Semantics: button, "Resume game".
4. 12dp gap.
5. **Sound toggle** — `OutlinedButton.icon`, full width, height 48dp. Icon: `Icons.volume_up` / `Icons.volume_off`. Label: "Mute" / "Unmute". Semantics: button, label reflects current state.
6. 12dp gap.
7. **End Game** — `TextButton`, full width, height 48dp. Label "End Game" in `Colors.red`. Tap: `pauseGame()` then `context.go('/')`. Semantics: button, "End game and return to home screen".

No confirmation dialog for "End Game" in Phase 4 — the 5-second pause before the tap is the grace period. (Phase 5 may add a confirm dialog.)

**Source:** Direct port of Flags `map_screen.dart` `_buildPauseOverlay()`.

### 7. Completion Screen

**Widget:** `CompletionScreen` extends `ConsumerStatefulWidget`, `SingleTickerProviderStateMixin`.

**Route:** `/complete` via `go_router`, receives `GameSession` as `extra`. Called after `gameSessionProvider` reaches `GamePhase.completed`.

**Background:** `#F5F5F5`.

**AppBar:** `backgroundColor` = mode color, `foregroundColor` = white, title = mode name. Leading: `Icons.home` IconButton navigates to `/`. No automatic back navigation (home is the only exit).

**Body:** `SingleChildScrollView(padding: symmetric(horizontal: 24, vertical: 32))` → `Column(crossAxisAlignment: center)`:

1. **Star row:** 3 `Icon` widgets, `Icons.star_rounded` / `Icons.star_outline_rounded`, 56dp, amber / `Colors.grey.shade400`. Horizontal padding 4dp each.
2. 8dp gap.
3. **Completion title:** "Well done!" — 28sp w700, mode color.
4. **Personal-best badge** (shown only when `_isNewPb`): 8dp gap + `Container(padding: symmetric(h:16, v:8), decoration: BorderRadius.circular(20), color: Colors.amber.shade700)` + "New Personal Best!" 14sp w700 white.
5. 32dp gap.
6. **Score card** (`RepaintBoundary` for future share; white card, BorderRadius 16, boxShadow black12 blur 8 offset (0,4), padding 24): four `_StatRow` rows (Score / Time / Mode / Previous Best) separated by `Divider(height: 24)`. "Previous Best" row hidden when `previousBest == null`.
7. 32dp gap.
8. **Primary CTA "Back to Menu"** — `ElevatedButton.icon`, full width, height 56dp, `BorderRadius.circular(14)`, icon `Icons.home`, backgroundColor = mode color, foregroundColor white, 17sp w700.
9. 12dp gap.
10. **Secondary CTA "Play Again"** — `OutlinedButton.icon`, full width, height 48dp, `BorderRadius.circular(14)`, icon `Icons.replay`, foregroundColor = mode color, borderSide = mode color.

No share button in Phase 4 (v2 scope, per D-13).

**PB confetti overlay:** `AnimatedBuilder` on `_pbController` (2000ms), `IgnorePointer`, `Opacity` fading to 0 after 80% progress. `_ConfettiPainter` — 40 particles, fixed seed (42), 6 colors (red/blue/green/yellow/purple/orange), circles radius 6dp. Shown as `Positioned.fill` in a `Stack`.

**Star formula (D-11, exact):**
- `previousBest == null` → 3 stars, `_isNewPb = false` (first game ever).
- `score < previousBest` → 3 stars, `_isNewPb = true`.
- `score <= (previousBest * 1.20).ceil()` → 2 stars, `_isNewPb = false`.
- else → 1 star, `_isNewPb = false`.

**Source:** Direct port of Flags `completion_screen.dart`. Omit: `share_plus`, AdMob interstitial, AdMob banner.

## Routing Contract

| Route | Widget | extra parameter |
|-------|--------|-----------------|
| `/` | `HomeScreen` | none |
| `/play/{modeName}` | `MapScreen` (game screen) | `GameMode` enum value |
| `/complete` | `CompletionScreen` | `GameSession` object |

`/play` route receives `GameMode` as a path parameter (`state.pathParameters['modeName']`) parsed to `GameMode.values.byName(...)`. Phase 3 registered `/play` without a parameter; Phase 4 changes it to `/play/:mode` or passes via `extra`. Use `extra: mode` (existing pattern in Flags `context.go('/play/${mode.name}', extra: {...})`).

Back-button guard: `onExit` callback on the `/play` GoRoute — if `GamePhase.playing`, call `pauseGame()` and show pause overlay; return `false` to suppress navigation. If `GamePhase.paused`, show pause overlay; return `false`. Otherwise allow navigation.
